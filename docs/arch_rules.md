# Архітектурні правила проєкту "Your Energy"

## 1. Технологічний стек

- **Мова:** JavaScript (ES6+). Використання модулів (`import`/`export`).
- **Збірка:** Vite (конфігурація у `vite.config.js` не змінюється без критичної
  потреби).
- **Стилі:** Чистий CSS. Використовуємо `@import` у `src/css/styles.css` для
  підключення файлів компонентів.
- **HTML:** HTML5. Використання плагіну `vite-plugin-html-inject` для
  підключення фрагментів з `src/partials/`.
- **Пакетний менеджер:** npm.

## 2. Заборони (Hard Constraints)

- **ЖОДНИХ Фреймворків:** Заборонено використання React, Vue, Angular, jQuery
  тощо.
- **ЖОДНИХ бібліотек UI:** (на кшталт Bootstrap, Tailwind, Material UI).
  Стилізація пишеться вручну.
- **Структура:** Не змінювати кореневу структуру папок шаблону
  `vanilla-app-template`.

## 3. Структура коду та файлів

- **JavaScript:**
  - Точка входу: `src/main.js`.
  - Логіка розбивається на модулі. Створити папку `src/js/` для скриптів
    (наприклад, `api.js`, `modal.js`, `filters.js`).
- **HTML Partials:**
  - Усі повторювані елементи (хедер, футер, модалки) зберігаються у
    `src/partials/`.
- **CSS:**
  - Для кожного компонента — окремий CSS файл у `src/css/` (наприклад,
    `header.css`, `exercises.css`).
  - Використовуємо методологію BEM (Block Element Modifier) для іменування
    класів.

## 4. Робота з даними та API

- **HTTP-клієнт:** Використовуємо вбудований `fetch` (або `axios`, якщо він
  встановлений у `package.json`, але пріоритет — Vanilla JS).
- **Асинхронність:** Використовуємо `async/await`.
- **Обробка помилок:** Блок `try/catch` для кожного запиту.
- **LocalStorage:**
  - Ключі: `quote`, `favorites`.
  - Обробка помилок парсингу JSON при читанні зі стораджу.

## 5. Стиль коду

- Слідувати налаштуванням `.prettierrc.json`.
- Коментарі до складних функцій.
- Змінні та функції називаються англійською мовою в `camelCase`.
