# Вимоги до проєкту "Your Energy"

## 1. Загальна інформація

Проєкт є односторінковим застосунком (SPA), розробленим для пошуку та перегляду
інформації про фізичні вправи.

- **Базовий URL API:** `https://your-energy.b.goit.study/api`
- **Формат відповідей:** JSON
- **Автентифікація:** Не вимагається (публічний API).
- **Стек:** HTML, CSS, Vanilla JavaScript (ES6+), Vite (збірка).
- **Деплой:** GitHub Pages.

## 2. Бізнес-логіка та сценарії користувача

### Головна сторінка (Home)

1. **Цитата дня:**
   - Відображається у лівій колонці або спеціальному блоці.
   - Запит на API виконується **один раз на добу**.
   - Дані (цитата + дата) кешуються в `localStorage`. Якщо дата не змінилася,
     беремо з кешу.
2. **Фільтри:**
   - Три вкладки: "Muscles" (за замовчуванням), "Body parts", "Equipment".
   - При зміні фільтра завантажується список категорій (пагінація
     підтримується).
   - Активний фільтр має візуальне виділення.
3. **Категорії та Вправи:**
   - При кліку на картку категорії список категорій приховується, завантажується
     список вправ цієї категорії.
   - Картка вправи містить: назву, рейтинг, спалені калорії/час, частину тіла,
     ціль, кнопку "Start".
4. **Пошук:**
   - Рядок пошуку фільтрує вправи за ключовим словом (`keyword`).
   - Пошук спрацьовує по події `submit`.
5. **Пагінація:**
   - Реалізована "серверна" пагінація (параметри `page`, `limit`).
   - Пагінація відображається як для списку категорій, так і для списку вправ.

### Модальне вікно вправи

- Відкривається при кліку на "Start".
- Відображає: відео, назву, рейтинг, характеристики, опис.
- **Функціонал:**
  - Додати/видалити з "Favorites" (синхронізація з `localStorage`).
  - Оцінити вправу (кнопка "Give a rating").

### Модальне вікно рейтингу

- Відкривається з модального вікна вправи.
- Містить: вибір оцінки (зірки/радіокнопки), email, коментар (опціонально).
- Валідація email перед відправкою.
- Відправка `PATCH` запиту на сервер.

### Сторінка "Favorites"

- Окремий маршрут (або віртуальна сторінка).
- Відображає список вправ, збережених у `localStorage`.
- Якщо список порожній — показувати повідомлення-заглушку.
- Картка в "Favorites" має кнопку видалення (смітник), яка миттєво прибирає
  картку з DOM і оновлює сховище.

### Футер (Footer)

- Містить форму підписки на розсилку.
- Валідація email.
- Відправка `POST` запиту.
- Посилання на соцмережі GoIT.

## 3. Технічні обмеження та API Endpoints

### Ключові Endpoints

| Метод | URL                      | Опис               | Параметри                                                             |
| ----- | ------------------------ | ------------------ | --------------------------------------------------------------------- |
| GET   | `/quote`                 | Цитата дня         | -                                                                     |
| GET   | `/filters`               | Отримання фільтрів | `?filter=Muscles&page=1&limit=12`                                     |
| GET   | `/exercises`             | Список вправ       | `?bodypart=...&muscles=...&equipment=...&keyword=...&page=1&limit=10` |
| GET   | `/exercises/{id}`        | Деталі вправи      | Path param: `id`                                                      |
| PATCH | `/exercises/{id}/rating` | Оцінка вправи      | Body: `{ rate: number, email: string, review: string }`               |
| POST  | `/subscription`          | Підписка           | Body: `{ email: string }`                                             |

### Робота з LocalStorage

- **Key `quote`:** `{ quote: string, author: string, date: string }` — для
  кешування цитати.
- **Key `favorites`:** `[{...exerciseData}, ...]` — масив об'єктів улюблених
  вправ.

### Обробка помилок

- У разі помилок API (4xx, 5xx) користувачеві має відображатися нотифікація
  (наприклад, через бібліотеку `iziToast` або кастомний тост), а не просто лог у
  консоль.
